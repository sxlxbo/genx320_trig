# genx320_trig
代码1(检测触发信号准备离职）使用步骤（树莓派 5 上操作）

运行代码：打开终端，执行命令：
python3 genx320_trigger.py


触发录制：给脉冲源发送触发信号（如按按键、发脉冲），代码会自动：
检测到触发信号（防抖处理，避免误触发）；
触发后录制 5 秒事件数据；
生成带时间戳的.raw 文件，保存在 ./trigger_records 目录；
录制完成后验证触发时间戳，确认触发生效。

关键定制化说明
修改录制时长：把 record_duration = 5 改为你需要的时长（如 10 秒则改 10）；
改为下降沿触发：把 TriggerInMode.ON_RISING_EDGE 改为 TriggerInMode.ON_FALLING_EDGE；
修改保存路径：把 save_dir = "./trigger_records" 改为你想保存的目录（如 /home/pi/records）。
常见问题排查
未检测到相机：检查转接板与树莓派 5 的连接、树莓派供电（需 15W 以上电源）；
触发无响应：
检查接线：脉冲源 Signal→J3-1，GND→J3-4，别接反；
确认脉冲源输出 3.3V/5V 电平，脉冲宽度≥1μs；
录制文件为空：确保触发后有动态场景（事件相机静态场景无数据）。

代码2使用前注意事项：
1. 安装依赖
树莓派上先安装 GPIO 控制库：
sudo apt update
sudo apt install -y python3-rpi.gpio

# 最重要的完整操作流程（一步步跟着做）
## 接线：按第一步的接线表接好树莓派和 GenX320；
## 打开两个终端窗口（树莓派上）：
### 终端 1：运行 GenX320 触发检测代码：
   # python3 genx320_continuous_trigger.pyy
### 终端 2：运行树莓派触发信号生成代码：
   # python3 rpi5_trigger_generator.py
## 选择触发模式：
      在终端 2 输入 1（按键触发），按回车；
      按 s 键，终端 2 会打印 “已发送触发脉冲”；
      终端 1 会立即检测到触发，开始录制 5 秒事件数据；
      
    终端 2 输入 2（选择自动触发模式），程序会进入自动触发流程：
    程序会先提示你输入自动触发的间隔时间（单位：秒），如果直接回车，就会使用默认的 2 秒。
    之后，树莓派会每隔你设定的时间，自动向 GenX320 发送一次触发脉冲。
    每次发送脉冲时，终端 2 会打印：🎉 树莓派已发送触发脉冲！时间：HH:MM:SS。
    同时，终端 1（运行 GenX320 触发录制代码）会立即检测到触发，自动开始录制 5 秒事件数据，并生成新的.raw文件。
这个过程会一直循环，直到你在终端 2 按 Ctrl+C 终止程序。
