# genx320_trig
代码1(检测触发信号准备离职）使用步骤（树莓派 5 上操作）

运行代码：打开终端，执行命令：
python3 genx320_trigger.py


触发录制：给脉冲源发送触发信号（如按按键、发脉冲），代码会自动：
检测到触发信号（防抖处理，避免误触发）；
触发后录制 5 秒事件数据；
生成带时间戳的.raw 文件，保存在 ./trigger_records 目录；
录制完成后验证触发时间戳，确认触发生效。

关键定制化说明
修改录制时长：把 record_duration = 5 改为你需要的时长（如 10 秒则改 10）；
改为下降沿触发：把 TriggerInMode.ON_RISING_EDGE 改为 TriggerInMode.ON_FALLING_EDGE；
修改保存路径：把 save_dir = "./trigger_records" 改为你想保存的目录（如 /home/pi/records）。
常见问题排查
未检测到相机：检查转接板与树莓派 5 的连接、树莓派供电（需 15W 以上电源）；
触发无响应：
检查接线：脉冲源 Signal→J3-1，GND→J3-4，别接反；
确认脉冲源输出 3.3V/5V 电平，脉冲宽度≥1μs；
录制文件为空：确保触发后有动态场景（事件相机静态场景无数据）。
